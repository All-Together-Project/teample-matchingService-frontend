## 1. 프로젝트 개요

### 서비스 소개

**ALL 투게더**는 프로젝트 팀 구성을 위한 통합 팀 매칭 플랫폼입니다. 프로젝트 공고 등록, 역할별 지원, AI 기반 팀원 추천, 상호 평가 시스템을 통해 신뢰도 높은 팀 빌딩 경험을 제공합니다.

### 핵심 가치

| 가치 | 설명                                                   |
| ---- | ------------------------------------------------------ |
| 신뢰 | 온도/티어 기반 신뢰도 시스템으로 검증된 팀원 매칭      |
| 효율 | AI 추천으로 관심사/역량에 맞는 프로젝트·팀원 자동 연결 |
| 소통 | 댓글·쪽지·시스템 알림으로 끊김 없는 커뮤니케이션       |

### 기술 스택

| 구분       | 기술                                                |
| ---------- | --------------------------------------------------- |
| 프론트엔드 | React 18, TypeScript, Vite, Zustand, TanStack Query |
| 백엔드     | Spring Boot (예정)                                  |
| 인증       | JWT (Access Token + Refresh Token)                  |
| 스타일     | CSS Modules                                         |

## 3. 기능 명세

---

### 3.1 회원 및 인증 관리

#### 3.1.1 회원가입

| 항목          | 내용                                                                    |
| ------------- | ----------------------------------------------------------------------- |
| **설명**      | 이메일 기반 회원가입                                                    |
| **접근 권한** | 비로그인 사용자                                                         |
| **입력 필드** | 이름(필수), 닉네임(필수), 이메일(필수), 비밀번호(필수), 약관 동의(필수) |

**비밀번호 정책**

- 최소 8자 이상
- 영문 + 숫자 + 특수문자 포함 필수
- 입력 시 강도 표시 (약/보통/강)

**유효성 검사**

- 이메일 중복 체크
- 닉네임 중복 체크
- 비밀번호 확인 일치 여부

**성공 시 동작**

- JWT 발급 후 자동 로그인
- 프로필 설정 페이지(`/signup/profile`)로 이동

---

#### 3.1.2 로그인

| 항목          | 내용                                                                             |
| ------------- | -------------------------------------------------------------------------------- |
| **설명**      | 이메일/비밀번호 기반 로그인                                                      |
| **인증 방식** | JWT (Access Token 30분, Refresh Token 7일)                                       |
| **토큰 저장** | `localStorage`                                                                   |
| **만료 처리** | Axios 인터셉터에서 Refresh Token으로 자동 갱신, 실패 시 로그인 페이지 리다이렉트 |

---

#### 3.1.3 프로필 설정 (회원가입 후 2단계)

3단계 스텝 구성:

**Step 1 — 기본 정보**

- 한 줄 소개 (최대 80자)
- 전공
- 학교/소속

**Step 2 — 활동 역할 선택**

- 개발 / 디자인 / 기획 / 데이터 / 마케팅 / 기타 (복수 선택)

**Step 3 — 관심 태그 선택**

- 기술 스택 태그 (TECH 타입, 최대 10개)
- 관심 분야 태그 (INTEREST 타입, 최대 10개)
- 선택한 태그 기반으로 AI 추천 엔진 가중치 데이터로 활용

---

#### 3.1.4 마이페이지

| 기능             | 설명                                             |
| ---------------- | ------------------------------------------------ |
| 프로필 조회      | 닉네임, 온도, 티어, 한 줄 소개, 전공, 역할, 태그 |
| AI 추천 프로젝트 | 관심 태그 기반 맞춤 프로젝트 최대 6개            |
| 받은 리뷰 목록   | 평가 항목별 점수 + 코멘트 타임라인               |
| 프로필 편집      | 한 줄 소개, 전공, 역할, 태그 수정                |

**신뢰도 지표**

- 온도 (0.0 ~ 100.0°) — 누적 평가 점수 기반 자동 산출
- 티어 — 활동 이력 + 평균 점수 기반 자동 갱신

---

#### 3.1.5 타 사용자 프로필 조회

- 공개 처리된 프로필만 조회 가능
- 표시 항목: 닉네임, 온도, 티어, 한 줄 소개, 역할, 태그, 리뷰 요약
- 쪽지 보내기 버튼 제공 (로그인 사용자만)

---

### 3.2 프로젝트 관리

#### 3.2.1 프로젝트 CRUD

| 기능      | 접근 권한     | 설명                             |
| --------- | ------------- | -------------------------------- |
| 목록 조회 | 전체          | 페이지네이션, 필터, 검색 지원    |
| 상세 조회 | 전체          | 공고 내용, 역할, 댓글, 지원 버튼 |
| 등록      | 로그인 사용자 | 신규 모집 공고 작성              |
| 수정      | 프로젝트 리더 | 공고 내용 수정                   |
| 삭제      | 프로젝트 리더 | 공고 삭제 (soft delete 권장)     |

**등록 입력 필드**

| 필드           | 필수 | 설명                                                         |
| -------------- | ---- | ------------------------------------------------------------ |
| 제목           | ✅   | 최대 100자                                                   |
| 카테고리       | ✅   | 웹 서비스/앱 개발/AI·ML/커머스/핀테크/헬스케어/에듀테크/기타 |
| 프로젝트 소개  | ✅   | 상세 설명 (마크다운 지원 권장)                               |
| 모집 인원      | ✅   | 2~20명                                                       |
| 시작 예정일    | -    | 날짜 선택                                                    |
| 종료 예정일    | -    | 날짜 선택                                                    |
| 기술 스택 태그 | ✅   | 최대 10개, AI 추천 가중치 및 필터에 활용                     |
| 모집 역할      | ✅   | 역할명, 역할 유형, 설명, 인원, 필요 태그                     |

---

#### 3.2.2 프로젝트 상태 관리

```
모집중(RECRUITING) → 모집완료(COMPLETED) → 종료(CLOSED)
```

| 상태       | 설명          | 가능 액션            |
| ---------- | ------------- | -------------------- |
| RECRUITING | 팀원 모집 중  | 지원 가능, 댓글 가능 |
| COMPLETED  | 팀 구성 완료  | 지원 불가, 댓글 가능 |
| CLOSED     | 프로젝트 종료 | 리뷰 작성 가능       |

- 상태 변경 권한: 프로젝트 리더만
- CLOSED 전환 시 팀원들에게 리뷰 작성 알림 발송

---

#### 3.2.3 프로젝트 멤버 관리

- 지원서 승인 시 자동으로 프로젝트 멤버로 등록
- 멤버 목록 조회 (프로젝트 상세 페이지)
- 리더에게 AI 추천 팀원 최대 5명 표시

---

### 3.3 팀 매칭 시스템

#### 3.3.1 역할별 지원서 시스템

**지원 가능 조건**

- 로그인 상태
- 프로젝트 상태 = RECRUITING
- 본인이 리더가 아닌 경우
- 동일 프로젝트 중복 지원 불가

**지원서 입력 필드**

| 필드           | 필수 | 제한                                 |
| -------------- | ---- | ------------------------------------ |
| 지원 역할 선택 | ✅   | 모집 역할 중 선택 (미달 역할만 표시) |
| 지원 동기      | ✅   | 최대 500자                           |
| 자기소개       | ✅   | 최대 500자                           |

---

#### 3.3.2 지원서 승인/거절 및 팀원 자동 등록

| 액션 | 처리 결과                                                         |
| ---- | ----------------------------------------------------------------- |
| 승인 | 지원자 상태 → APPROVED, 프로젝트 멤버 자동 등록, 시스템 쪽지 발송 |
| 거절 | 지원자 상태 → REJECTED, 시스템 쪽지 발송                          |

- 승인 시 해당 역할의 `filledCount` 자동 증가
- 역할 인원이 모두 채워지면 해당 역할 비활성화

---

#### 3.3.3 지원 내역 및 상태 실시간 추적

**마이페이지 > 지원 내역 탭**

| 상태             | 표시 색상 | 가능 액션      |
| ---------------- | --------- | -------------- |
| PENDING (검토중) | 주황      | 지원 취소 가능 |
| APPROVED (승인)  | 초록      | -              |
| REJECTED (거절)  | 빨강      | -              |
| CANCELLED (취소) | 회색      | -              |

### 3.4 리뷰 시스템

#### 3.4.1 팀원 상호 평가

**작성 조건**

- 프로젝트 상태 = CLOSED
- 해당 프로젝트의 팀원인 경우
- 본인을 제외한 팀원에게만 작성 가능
- 프로젝트당 1인 1회만 작성

**평가 항목 (각 1~5점)**

| 항목                      | 설명                    |
| ------------------------- | ----------------------- |
| 전문성 (Expertise)        | 해당 역할의 기술적 역량 |
| 소통성 (Communication)    | 팀원과의 원활한 소통    |
| 시간 준수성 (Punctuality) | 마감 일정 준수 여부     |
| 참여 태도 (Participation) | 회의 참석, 적극적 기여  |
| 열정 (Passion)            | 프로젝트에 대한 몰입도  |

- 코멘트 (선택, 최대 300자)

---

#### 3.4.2 리뷰 기반 신뢰도 갱신

- 리뷰 등록 시 해당 사용자의 온도/티어 자동 재계산
- 온도 계산식: `(전체 평가 항목 평균 / 5) × 100` 에 가중치 적용
- 티어는 누적 리뷰 수 + 평균 점수 기준으로 자동 상향

---

#### 3.4.3 멤버 리뷰 상세 및 요약 조회

**요약 (ReviewSummary)**

- 항목별 평균 점수 (막대 그래프)
- 전체 평균 및 총 리뷰 수
- 가장 최근 코멘트 1건

**상세 목록**

- 프로젝트명, 작성자, 날짜, 항목별 점수, 코멘트

---

### 3.5 소통 및 알림 (쪽지 시스템)

#### 3.5.1 시스템 자동 쪽지

| 트리거        | 발송 내용                                                     |
| ------------- | ------------------------------------------------------------- |
| 지원서 승인   | "프로젝트 [제목]에 팀원으로 합류하셨습니다!"                  |
| 지원서 거절   | "프로젝트 [제목] 지원이 아쉽게도 거절되었습니다."             |
| 프로젝트 종료 | "프로젝트 [제목]이 종료되었습니다. 팀원 리뷰를 작성해주세요." |

- 발신자: 시스템 (type = SYSTEM)
- 읽음 처리 대상 아님

---

#### 3.5.2 1:1 개인 쪽지

| 기능        | 설명                                         |
| ----------- | -------------------------------------------- |
| 쪽지 보내기 | 유저 프로필 페이지에서 발송, 내용 최대 500자 |
| 수신함      | 받은 쪽지 목록, 읽지 않은 쪽지 카운트 표시   |
| 발신함      | 보낸 쪽지 목록                               |
| 읽음 처리   | 쪽지 선택 시 자동 읽음 처리                  |

### 3.6 댓글

#### 3.6.1 프로젝트 공고 댓글

| 기능      | 접근 권한     | 설명                          |
| --------- | ------------- | ----------------------------- |
| 댓글 작성 | 로그인 사용자 | 프로젝트 상세 하단            |
| 댓글 조회 | 전체          | 최신순 정렬                   |
| 댓글 삭제 | 작성자 본인   | Soft delete 권장              |
| 답글 작성 | 로그인 사용자 | 특정 댓글에 답글 (계층 1단계) |

**계층 구조**

```
댓글 (parentId = null)
  └── 답글 (parentId = 댓글 id)
```

- 답글에 대한 재답글은 허용하지 않음 (1단계 계층만)

### 3.7 검색 및 AI 추천

#### 3.7.1 필터 검색

| 필터 항목      | 설명                            |
| -------------- | ------------------------------- |
| 키워드         | 제목, 설명 내 텍스트 검색       |
| 상태           | RECRUITING / COMPLETED / CLOSED |
| 카테고리       | 서비스 분야별 필터              |
| 기술 스택 태그 | 복수 선택 가능 (AND 조건)       |
| 관심 분야 태그 | 복수 선택 가능                  |

---

#### 3.7.2 사용자 맞춤형 프로젝트 추천

- 로그인 사용자의 기술 태그, 관심 분야 태그, 역할 데이터 기반
- 유사도 점수가 높은 RECRUITING 상태 프로젝트 최대 6개 반환
- 마이페이지 상단 노출

---

#### 3.7.3 프로젝트 맞춤형 팀원 추천

- 프로젝트 상세 페이지 (리더에게만 노출)
- 모집 역할에 필요한 태그와 사용자 프로필 매칭
- 티어/온도 가중치 적용 상위 5명 추천
- 추천 유저 프로필 클릭 시 해당 프로필 페이지 이동

### 주요 화면 상세

#### 프로젝트 목록 (`/projects`)

**상단 영역**

- 페이지 제목 + 모집 공고 등록 버튼 (우측)

**필터 영역**

- 키워드 검색 인풋
- 상태 탭 (전체 / 모집중 / 모집완료 / 종료)
- 기술 스택 태그 멀티 선택 칩

**목록 영역**

- 카드 그리드 (3열 기준)
- 카드 정보: 상태 뱃지, 카테고리, 제목, 설명 2줄 요약, 태그 3개, 리더 닉네임, 현재/최대 인원

**하단 영역**

- 페이지네이션 (이전/다음 버튼 + 현재 페이지/전체 페이지)

---

#### 프로젝트 상세 (`/projects/:id`)

**2단 레이아웃** (메인 + 우측 사이드바)

**메인 영역**

1. 헤더: 상태 뱃지, 카테고리, 제목, 리더 정보, 등록일, 태그
2. 프로젝트 소개 섹션
3. 모집 역할 카드 목록 (역할명, 현재/최대 인원, 설명, 필요 태그)
4. 댓글 섹션

**사이드바**

- 지원하기 버튼 (비리더 + RECRUITING 상태일 때만 표시)
- 리더에게: 공고 수정 / 삭제 버튼
- 프로젝트 기본 정보 (상태, 시작일, 종료일, 모집 인원)
- AI 추천 팀원 (리더에게만)

## 7. 비기능 요구사항

### 성능

| 항목                      | 목표            |
| ------------------------- | --------------- |
| 페이지 초기 로딩          | 3초 이내        |
| API 응답 시간             | 평균 500ms 이내 |
| 프로젝트 목록 페이지 크기 | 기본 12개       |

### 보안

- 모든 API는 HTTPS 통신
- JWT Access Token 만료 시간: 30분
- JWT Refresh Token 만료 시간: 7일
- 비밀번호 bcrypt 암호화 저장
- 민감한 API는 서버 사이드 권한 검증 필수

### 확장성

- 태그 시스템은 관리자 페이지에서 동적 추가 가능하도록 설계
- AI 추천 엔진은 별도 마이크로서비스로 분리 고려
- 이미지 업로드는 CDN(S3 등) 연동 예정
